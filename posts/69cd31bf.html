GraphQL 学习笔记

前言

1. 什么是 GraphQL

GraphQL是一个用于API的查询语言，是一个使用基于类型系统来执行查询的服务端运行时（类型系统由你的数据定义）。GraphQL并没有和任何特定数据库或者存储引擎绑定，而是依靠你现有的代码和数据支撑。

  • 为什么使用GraphQL

    • 灵活的数据获取: 客户端可以请求确切的数据结构，避免了过度和不足的数据获取问题。
    • 单一端点: 所有的查询都通过一个单一的端点进行，这简化了客户端与服务器的交互。
    • 强类型系统: GraphQL使用强类型定义API，这有助于在开发过程中捕获错误，并提供更好的工具支持，如自动生成文档和代码。
    • 实时数据更新: 通过订阅subscriptions，GraphQL支持实时数据更新，使得客户端可以在数据发生变化时自动接收更新。

  • 与REST的对比 (优点)

    • 避免多次请求: 在REST中，客户端可能需要多次请求不同的资源来获取相关的数据。在GraphQL中，可以在单一查询中请求所有相关的数据。
    • 精确的数据获取: 客户端可以明确指定所需的数据字段，避免了获取不需要的数据。这在网络不稳定或带宽有限的环境中特别有用。

  • 与REST的对比 (缺点)

    • 复杂的查询解析: 由于GraphQL查询的灵活性，解析和执行查询可能比REST更加复杂，尤其是在复杂的业务逻辑和大规模数据集的情况下。
    • 缓存困难: 与REST不同，GraphQL查询不是固定的URL，所以标准的HTTP缓存机制难以直接应用。需要特别的缓存策略来应对这一问题。
    • 学习曲线: 这个就不用说了 😅

2. 字段 Fields

在GraphQL中，字段fields是定义在类型type上的属性，它们表示该类型包含的数据。每个字段都有一个名称和一个返回类型，可以是标量类型（如 String、Int）、对象类型、枚举类型、列表类型等。字段还可以接受参数，这些参数允许在查询时提供附加信息或过滤数据。

  • 在这个示例中 (其实也可以理解为在这里定义了schema)

    • User类型有三个字段: id、name和age。这些字段定义了User类型包含的数据。
    • Query 类型有一个字段: user。这个字段表示一个查询操作，可以根据用户的id来获取User类型的数据。

  • 什么是schema 😐

    • 类型定义Type Definitions: 定义了API中的各种类型，包括标量类型、对象类型、枚举类型等。
    • 根类型Root Types: 通常包括Query、Mutation和Subscription，它们分别定义了可用的查询、变更和订阅操作。(客户端可以调用的操作的入口点)

  • 此外，字段还可以包含解析器resolver，它是一个函数，用于告诉GraphQL如何获取该字段的值。当客户端请求某个字段时，GraphQL会调用对应的解析器来获取数据。

  • 最后，从客户端发送一个查询请求，指定它想要获取的字段。

  • 所以实际上整个流程是: 先定义schema，然后写resolver，最后在客户端进行请求

3. 参数 Arguments

在GraphQL中，参数Arguments是用来向字段和操作（如查询和变更）传递输入数据的。参数允许客户端在请求中提供额外的信息，用于筛选、过滤、或指定要操作的数据。

  • 参数通常定义在GraphQL schema中的字段上，可以与标量类型、输入对象类型等关联。参数可以是可选的，也可以是必需的（通过添加!符号来表示）。

在GraphQL中，方括号[]用于表示一个列表类型。

  • user 字段有一个必需的id参数，它用于指定要查询的用户。
  • users 字段有一个可选的age参数，用于筛选特定年龄的用户。

在变更Mutation中，参数用于提供操作所需的输入数据。例如，创建一个新用户时，需要提供用户的详细信息

  • 总结
    • 参数Arguments用于向字段和操作传递输入数据。
    • 参数可以是必需的或可选的。
    • 参数类型可以是标量类型或输入对象类型。
    • 在解析器中，参数通过args对象传递，可以用来筛选和操作数据。

4. 别名 Aliases

在GraphQL中，别名Aliases是一种用于重命名查询结果中字段名称的功能。它允许客户端在查询同一个字段多次时，使用不同的名字来区分这些字段的返回结果，或简化返回数据中的字段名称。

  • 避免命名冲突: 当查询结果包含多个同名字段时，别名可以避免命名冲突。例如，当你需要查询同一资源的多个实例时，可以使用别名来区分它们。
  • 自定义响应格式: 别名允许客户端自定义返回数据中的字段名称，这样可以使数据格式更符合客户端的需求。

假设有一个查询可以获取用户信息，需要查询两个不同用户的详细信息，可以使用别名来区分它们。

在这个查询中，user1和user2是别名，它们分别对应user字段的两个不同调用。id、name和 age是查询返回的用户信息字段。

在返回的结果中，user1和user2成为响应数据中的字段名称，分别包含两个不同用户的信息。

5. 片段 Fragments

片段Fragments是GraphQL-中的一种机制，用于在多个查询中重用一组字段选择。片段允许你将一些常用的字段定义在一个片段中，然后在查询中引入这个片段，从而避免重复定义字段。这不仅提高了查询的可读性和维护性，还减少了冗余。

  • 片段定义了一个可以在查询中重用的字段集合。片段的语法如下
  • FragmentName是片段的名称，TypeName是片段应用的类型，field1, field2, ...是片段中包含的字段。

假设你有一个User类型，并且你经常需要查询用户的id、name和age字段。你可以定义一个片段来重用这些字段。

  • 嵌套片段
    • 片段可以嵌套使用，即在一个片段中引用另一个片段。可以更灵活地组织字段集合。
  • 片段的作用
    • 避免字段重复: 当多个查询或变更操作需要相同的一组字段时，片段可以帮助避免重复定义这些字段。
    • 提高查询可读性: 片段使得查询结构更加清晰和简洁，便于理解和维护。
    • 便于维护: 当需要修改某个字段集合时，只需要修改片段的定义，而不需要逐一修改每个查询。

6. 操作名称 Operation Name

在GraphQL中，操作名称Operation Name是一个可选的名称，用于标识特定的查询、变更Mutation或订阅Subscription操作。

  • 操作名称紧跟在操作类型之后。操作名称只能包含字母、数字和下划线，并且不能以数字开头。

在这个示例中，GetUser是操作名称，它标识了这个查询操作的具体用途。

7. 操作类型 Operation Type

在GraphQL中，操作类型Operation Type指的是你可以在API请求中执行的操作种类。主要有三种操作类型: 查询Query、变更Mutation和订阅Subscription。每种操作类型有不同的用途和特性，分别用于读取数据、修改数据和监听实时数据更新。

  • 查询 Query
    • 查询是最常用的操作类型，用于读取和获取数据。查询操作不会改变服务器上的数据，只会返回所请求的数据。它类似于REST API中的GET请求。
  • 变更 Mutation
    • 变更用于修改服务器上的数据，包括创建、更新或删除操作。与查询不同，变更操作可以改变服务器上的数据状态。它类似于REST API中的POST、PUT、PATCH和DELETE请求。
  • 订阅 Subscription
    • 订阅用于监听服务器上的事件，并在事件发生时实时接收更新。它通常用于需要实时数据更新的应用场景，如实时聊天、股票价格更新等。订阅操作会持续存在，一旦数据发生变化，服务器就会推送更新给客户端。

在这个例子中，subscription表示这是一个订阅操作，它监听新的用户添加事件。当有新用户被添加时，客户端会接收到包含新用户信息的更新。

8. 订阅 Subscription

订阅Subscription是GraphQL中的一种操作类型，它允许客户端订阅服务器上的事件，并在事件发生时实时接收数据更新。这种机制特别适合用于需要实时数据更新的应用场景，如实时聊天、实时通知、股票价格更新等。

  • Subscription 的工作原理
    • 建立订阅连接: 客户端发送一个订阅请求到服务器，订阅某个事件或数据。
    • 保持连接: 服务器和客户端之间保持一个长连接（通常是通过WebSocket实现）。
    • 事件触发: 当服务器上的数据发生变化时，触发相关事件。
    • 数据推送: 服务器将更新的数据推送给所有订阅了该事件的客户端。

WebSocket是一种协议，用于在客户端和服务器之间建立双向通信的持久连接。

9. 变量 Variables

在GraphQL中，变量Variables是一种用来动态传递参数给查询、变更Mutation或订阅Subscription操作的机制。使用变量可以使查询更灵活和可重用，并且可以避免在查询字符串中直接嵌入用户输入的数据，从而提高安全性和可读性。

在这个示例中，useQuery Hook用于执行GET_USER查询，并传递变量值{ id: userId }。userId是从组件的props中传递的。

变量的类型必须与GraphQL schema中定义的字段参数类型相匹配。例如，如果字段参数是String类型，那么相应的变量也必须是String类型。

10. 指令 Directives

指令Directives是GraphQL中的一个强大功能，用于在查询、变更Mutation或订阅Subscription中，动态地影响执行行为或返回结果。指令可以应用于字段、片段等，允许客户端或服务器在执行查询时调整行为，例如条件包含或排除字段、动态修改返回结果等。

  • GraphQL规范中定义了两个内置指令
    • @include: 条件包含
    • @skip: 条件跳过

@include指令用于根据条件动态包含某个字段或片段。它接受一个名为if的参数，该参数的值是一个布尔类型，决定该字段是否应包含在结果中。

  • 在这个示例中，如果变量$withAge的值为true，那么age字段将包含在结果中。

@skip指令与@include相反，它用于根据条件跳过某个字段或片段。它同样接受一个名为if的布尔类型参数。

  • 在这个示例中，如果变量$withoutAge的值为true，那么age字段将被排除。

除了内置指令，GraphQL还允许开发者定义自定义指令。这些指令可以用于更多的特定场景，如字段验证、格式化输出等。自定义指令需要在schema中定义，并提供相应的实现逻辑。

  • directive: 这是声明一个指令的关键词，用于定义新的指令。
  • @upper: 这是指令的名称。自定义指令的名称必须以@开头，在使用时也会用到这个符号。
  • on FIELD_DEFINITION: 这是指令的适用位置，指定了这个指令可以应用于哪些地方。在这个例子中，FIELD_DEFINITION表示该指令可以应用于字段定义上。

11. 查询 Query

查询Query是GraphQL中的一种操作类型，用于从服务器获取数据。它类似于传统REST API中的GET请求，但比其更加灵活和强大。

  • 在GraphQL schema中，查询类型通常放在Query类型中。Query类型描述了所有可以执行的查询操作。
  • 查询的特点
    • 精确的数据获取: 客户端可以指定所需的数据字段，避免不必要的数据传输。
    • 单一请求: 查询通常通过单一请求获取所有需要的数据，即使这些数据来自多个资源。

12. 变更 Mutations

变更Mutations是GraphQL中的一种操作类型，用于对服务器端的数据进行修改操作。与查询Query只用于获取数据不同，变更可以创建、更新或删除数据。每个变更操作都可以定义所需的输入参数和返回的结果结构。

  • 在GraphQL schema中，你需要定义变更类型。变更类型通常放在Mutation类型中，它描述了所有可以执行的数据修改操作。
  • 变更的解析器resolvers定义了如何执行这些数据修改操作。解析器通常与数据库或其他持久层交互来修改数据。
  • 变更的特点
    • 有副作用: 与查询不同，变更操作会改变服务器端的数据状态，因此具有副作用。
    • 通常返回数据: 尽管变更的主要目的是修改数据，它们通常也返回修改后的对象或状态，以便客户端确认操作结果。

13. 内联片段 Inline Fragments

内联片段Inline Fragments是GraphQL中的一种机制，允许在查询中根据对象的实际类型动态选择字段。它们主要用于处理接口Interface或联合类型Union type，当查询的对象类型不确定时，可以根据具体的类型返回不同的字段集。

  • 使用内联片段的场景

    • 接口和联合类型: 当查询的字段可能返回不同的类型时，内联片段允许你根据每种可能的类型指定不同的字段选择。
    • 类型检查: 内联片段可以用于在查询中执行类型检查和条件字段选择。

  • 内联片段的语法

    • 内联片段使用...后跟随on TypeName，然后是在{}中指定该类型的字段。

假设有一个接口Person，有两个实现类型Employee和Customer。我们希望在查询时根据实际的类型返回不同的字段。

总而言之，内联片段是GraphQL中用于处理接口和联合类型的一种机制，它允许开发者根据实际对象的类型选择不同的字段。这使得查询更加灵活和类型安全，能够根据返回数据的实际类型提供适当的字段。
