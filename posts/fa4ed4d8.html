知识点

1. AWS Cloud Overview

  • Global Services: IAM，Route53，CloudFront，WAF

AWS Regions

  • A region is a cluster of data centers
  • 考虑 Compliance (合规)，Proximity (临近用户)，Available Services 和 Pricing

AWS Availability Zone

  • 一个 Region 可以有 3 - 6 个 AZ
  • 每个 AZ 都是分开的

AWS Edge Locations

  • Deliver content to user with lower latency (离用户越近，Deliver 速度越快)

2. IAM

AWS IAM

  • Global Service
  • Root Account: Created by default, shoudn’t be shared
  • Users: People within your orgainzation, can be grouped
  • Groups: Only contain Users, not other Groups
    • User could belong to multiple Groups

IAM Permissions

  • 定义了 User 或者 Group 的 Permission
  • 注意 Least privilege principle

IAM Polices

  • Statements 里面必须包含的: Effect，Principal，Action，Resource
  • Inline Policy 是 Assign 给个人的


IAM MFA

  • Protect Root Accounts and IAM Users
  • MFA options: Virtual MFA Device, U2F Sercurity Key (USB)
  • 如果不用 MFA, 也可以用 Password Policy, 比如定义 Password 长度和类型

IAM CLI & SDK

  • 可以用 AWS Management Console, AWS CLI, AWS SDK 来 Access AWS
  • AWS CLI, AWS SDK 由 Access Keys 保护
  • CLI: 类似终端
  • SDK: 用在代码里面, Language-specific


IAM Roles

  • 授权 AWS 服务去做某些事
  • 比如 EB: 需要 EC2 Role 和 Service Role

IAM Security Tools

  • IAM Credentials Report: 返回用户数据和 Credentials
  • IAM Access Advisor: 显示当前用户被授权的服务和使用时间

IAM Best Practices

  • 使用 MFA
  • 不要分享 Root Account, Access Keys
  • 使用 Roles 来给 AWS 服务授权
  • 使用 IAM credentials Report 和 IAM Access Advisor 审核账户的权限

Summary

3. EC2 Fundamentals

AWS EC2

  • 属于 IaaS (Infrastructure as Service) 模型
  • User Data: EC2 boost 的时候会跑的代码 (比如往 EC2 Instance 里面装 Appache)
  • 绑定 AZ


EC2 Instance Type

  • General Purpose: Balanced
  • Compute Optimized: High performance
  • Memory Optimized: Process data in memory
  • Storage Optimized: Read and write data on local storage




EC2 Security Groups

  • Control how traffic is allowed into or out of EC2 instance
  • Security groups are “firewall” on EC2 instance
  • Can be attached to multiple instances
  • Locked down to a region / VPC


EC2 Instance Purchasing Options

  • On-Demand Intance: Short workload, pay by second (Linux 和 Windows)
  • Reserved: 1 - 3 years (72% discount，Reserve Capacity in an AZ)
    • Reserved Instance: Long workload
    • Convertible Reserved Instances (RI): Long workload with flexible instance (66% discount，但是可以自由改变 Instance Type)
  • Saving Plans: 1 - 3 years, commitment to an amount of usage (72% discount, 要定一个上限, 超过的就变成 On-Demand)
  • Spot Instance: Short workload, can lose instance (最便宜，90% discount)
  • Dedicated Host: Have access to the physical server (最贵)
  • Dedicated Instance: Your own instance on your own hardware
  • Capacity Reservation: Reserve capacity in AZ

EC2 Spot Instance

  • 只能在 Open, Active 或者 Disabled 的状态下 Cancel Spot Instance
  • 需要先 Cancel Spot Instance，然后再 Terminate (Cancel 不代表 Terminate)
  • Spot Fleets: Automatically request Spot Instance with lowest price (Spot + On-Demand)
    • 拿到最便宜的 Spot Instances


4. EC2 SAA Level

Elastic IP

  • Public IP: Unique across the whole web
  • Private IP: Unique across the private network, but two different private networks can have same IP (如下图所示)
  • Elastic IP: Fixed Public IP for EC2 instance
    • 为什么要 Elastic IP，因为 EC2 的 Public IP 会在重启后改变


EC2 Placement Groups

  • Cluster: Low-Latency, Single AZ, 快但有风险 (因为 AZ 可能会崩掉)
  • Spread: High Availability, Critical Application, 风险小但有限制, 每个 AZ 最多 7 个 Instance
  • Partition: Spreads instances across many different partitions within an AZ, 7 partitions per AZ, up to 100 instances (Big Data, Kafka, 风险小，限制低)




Elatic Network Interfaces (ENI)

  • 绑定 AZ
  • 处理 EC2 Instance 故障转移 (failover)

EC2 Hibernate

  • EC2 Hibernate: Make EC2 instance boot faster (Instance 不是 Stoped, 而是 Hibernated)
  • EC2 Instance Root Volume type must be an EBS volume

5. EC2 Instance Storage

AWS EBS

  • 绑定 AZ
  • Block-level storage
  • Can detached from an EC2 instance and attach to another
  • Provisioned Capacity (需要考虑到底要用多少)


EBS Snapshots

  • Make a backup (snapshot) of EBS volume
  • 可以将这个 Snapshot 用在其他 AZ 或者 Region 上
  • Stored incrementally, billed only for the changed blocks stored


AWS AMI

  • 可以更好的 Configure 和启动 EC2 Instance (Customization)
  • Built for a specific region and can copy across regions (不是直接在另一个 Region 用)


EC2 Instance Store

  • 物理硬盘
  • Block-level storage
  • Fast I/O performance, Good for cache
  • EC2 Instance Store lose when stopped (ephemeral, not for long-term)

EBS Volume Types

  • General Purpose SSD: Balanced, Cost-Effective
  • Provisioned IOPS SSD: Critical Application & Database
  • Hard Disk Drives (HDD)



EBS Multi-Attach

  • Attach same EBS volume to multiple EC2 instance in same AZ
  • Only for io1 / io2 family

EBS Encryption

  • EBS Encryption (三个方面)
    • Data at rest is encrypted
    • Data in flight is encrypted
    • Snapshot is encrypted
  • 首先 Create Snapshot，然后 Encrypt Snapshot，然后创建一个新的 EBS Volume (从 Encrypted Snapshot)， 然后将这个 Volume Attach 到原来的 Instance 上


AWS EFS

  • 可以 mount 到多个 EC2 Instance
  • Scale automatically, Pay-Per-Use, No Capacity Planning (Expensive)
  • EC2 Instances can access files on an EFS file system across AZs, Regions and VPCs


EFS vs EBS

  • EBS can only be attached to a single instance in the same AZ
  • EFS can be mounted on instances across multiple AZ


6. ELB & ASG

Scalability & High Availability

  • Vertical Scale: increase the size (但是有 hardware limit)
  • Horizontal Scale: increase the number
  • High Availability: running application in at least 2 AZ (disaster recovery)

AWS ELB

  • Foward traffic to multiple servers (eg. EC2)
  • High Availability across AZs, Handle failures of downstream instances
  • NLB provide static DNS and static IP, ALB only provide static DNS
  • Health Check: 可以知道对应的 EC2 Istance 是否是健康的

  • Application Load Balancer (HTTP/HTTPS)
  • Network Load Balancer (Allow TCP/UDP)
  • Gateway Load Balancer (Security)
  • Classic Load Balancer (deprecated in 2023)

Application Load Balancer (ALB)

  • Deal with Layer 7 (HTTP / HTTPS)
  • Micro services & Container-based application (Docker & ECS)
  • Route base on Path (/Home) or Query (?Platform=Mobile)
  • ALB adds an additional header called “X-Forwarded-For” contains the client’s IP address



Network Load Balancer (NLB)

  • Deal with Layer 4 (TCP / UDP)
  • One static IP per AZ or Use Elastic IP (Fixed IP V4)


Gateway Load Balancer (GWLB)

  • Deal with Layer 3, for Security
  • Use Geneve protocol on port 6081

Sticky Sessions (Session Affinity)

  • Same client is always redirect to the same instance behind a load balancer
  • Make sure user doesn’t loss session data

  • Application-based Cookies
    • Don’t use AWSALB, AWSALBAPP, AWSALBTG
  • Duration-based Cookies

Cross Zone Load Balancing

  • ELB distributes traffic evenly across all registered EC2 instances in all AZs
  • 所有 Instance 分到的 Traffic 一定是一样的
  • ALB Enabled By Default，NLB Disabled By Default

SSL Certificates (Enable HTTPS)

  • SSL Certificate: Allow traffic between client and load balancer encrypted in transit
  • Manage SSL certificates using ACM (AWS Certificate Manager)
  • Could be used in ALB & NLB, use SNI to make it work



Connection Draining

  • Waiting for existing connections to complete

AWS ASG

  • 可以和 ELB 一起用，来 Scale Up 或者 Down 基于服务器压力
  • 可以自动 Replace Unhealth Instance (和 ELB 一起用)
  • 可以保证最少或最多有多少 EC2 Instance 在工作
  • Free, pay for the underlying EC2 Instance


  • Launch Template: 可以作为 Template 来生成 EC2 Instance

  • 可以和 CloudWatch Alarm 一起根据服务器压力 Scale Out 或 Scale In

Scaling Policies

  • Dynamic Scaling:
    • Target Tracking Scaling, Simple / Step Scaling, Scheduled Actions
  • Predictive Scaling:
    • Forecast load and schedule scaling ahead


7. AWS Fundamentals

AWS RDS

  • 使用 RDS 的好处 (对比将数据库存在 EC2 上):
    • Automated provisioning, Scaling capability
    • Read Replicas 和 Multi-AZ
    • Backup
  • Amazon RDS RIs: 类似 Amazon EC2 RIs
  • Supports MySQL, PostgreSQL, MariaDB, Oracle, MS SQL Server, and Amazon Aurora
  • 可以有最多 15 个 Read Replicas

  • Storage Auto Scaling: For application with unpredictable workloads

Read Replicas vs. Multi AZ

  • Read Replicas: Scalability
  • Within AZ, Cross AZ or Cross Region
  • Async Replication
  • Read Replicas in Same Region is free (Replication in AZs), Cross Region is not


  • Multi AZ: Disaster Recovery
  • Sync Replication
  • Read Replicas can be setup as Multi AZ
  • Don’t need to change SQL connection string


RDS Custom

  • Could access the underlying database and OS (Only Oracle and Microsoft SQL)

AWS Aurora

  • Support Postgres and MySQL
  • Aurora storage automatically grows
  • High Availiability & Read Scaling (6 copies across 3 AZ)
    • One Master Write, Multiple Replicas



Aurora Advanced Concepts

  • Replicas Auto Scaling: 和 EC2 的 Auto Scaling 类似
  • Custom Endpoints
  • Aurora Serverless
  • Global Aurora: Cross Region replication takes less than 1 second
    • 同时保证至少有一个 Replica 可以用
  • Aurora Cloning: 可以快速 Access Aurora DB
  • Aurora Machine Learning





Backup & Monitoring

  • RDS Backups
    • Automated backups
    • Manual DB Snapshots

  • Aurora Backups
    • Automated backups
    • Manual DB Snapshots

  • RDS & Aurora Restore options

  • Aurora Database Cloning

RDS Security

  • At-rest encryption: using AWS KMS
  • In-flight encryption: using AWS TLS root certificates

RDS Proxy

  • Have a proxy for access RDS
    • Allow apps to pool and share DB connections (提高 DB Efficiency)
  • Serverless, Autoscaling, Highly Avaliable (multi-AZ)
  • Reduced RDS & Aurora failover time by up 66%
  • Not publicly accessible (Need to use VPC)

AWS ElastiCache

  • Manage Redis or Memcached (它的基本创建选项就是这两个, 应该是只为它们服务)
    • Cache are in-memory database with high performance and low latency
    • Cache helps reduce load off of database for read intensive workload
  • Help to make application stateless (AWS 负责大部分工作)

  • DB Cache: Get data from ElastiCache
    • If not avaliable, get from RDS and store in ElastiCache

  • User Session: Write Session data into ElastiCache
    • User in another instance could still be logged in

  • Redis vs Memcahed
    • Redis: Read Replicas, High Availiablity, Backup
    • Memcached: Not High Availiablity, No Backup, Have Risk to Lose Data

ElastiCache SAA Level

  • ElastiCache supports IAM Authentication for Redis
  • Memcached supports SASL-based authentication
  • IAM Database Authentication does not support Oracle

  • 3 patterns for ElastiCache
    • Lazy Loading, Write Through, Session Store

  • ElastiCache Use Case
    • Gaming Leaderboards (用 Redis 的 Sorted Sets)

8. Route 53

What’s DNS

  • Translate hostname into IP address
    • www.google.com -> 172.217.18.36



AWS Route 53

  • User can update the DNS records
  • Route 53 is a Domain Registrar

  • Route 53: Records
    • A: IPv4
    • AAAA: IPv6
    • CNAME: Map hostname to another hostname
    • NS: Name Servers


  • Route 53: Hosted Zones
    • Public Hosted Zones: public domain name
    • Private Hosted Zones: private domain name (VPC)


Route 53 TTL

  • TTL: Time To Live
    • how long the value should be cached (ELB 可能不会立即更新)

CNAME vs Alias

  • CNAME: Point a hostname to another hostname
  • Alais: Point a hostname to an AWS Resource
    • Targets: ELB, CloudFront, API Gateway 等等


Health Checks

  • Health Checks are only for public resources
  • Health Checks == Automated DNS Failover

Routing Policy (Simple)

  • Define how Route 53 responds to DNS queries
  • 7 种 Routing Policies

  • Simple: Route traffic to a single resource
    • If multiple values returned, a random one is chosen by client

Routing Policy (Weighted)

  • Weighted: Control the % of the request go to each resource

Routing Policy (Latency)

  • Latency: Redirect to the resource that has the least latency close to us
  • Latency is based on traffic between users and AWS Regions

Routing Policy (Failover)

  • Failover: Have a Primary Instance and a Secondary Instance
    • When failover, switch to Secondary Instance

Routing Policy (Geolocation)

  • Geolocation: Routing based on user location
    • Need to create a “Default” record
    • Use cases: website localization, restrict content distribution 等等

Routing Policy (Geoproximity)

  • Geoproximity: Route traffic to resource based on geolocation
    • But have the ability to shift more traffic to resource based on bias

Routing Policy (IP-Based)

  • IP-Based: Routing based on client’s IP address
    • Provide a list of CIDRs for your client

Routing Policy (Multi-Value)

  • Multi-Value: Route traffic to multiple resources
    • Not a subsitute for ELB

10. AWS S3

AWS S3

  • S3 Buckets:
    • Store objects (files) in “buckets”
    • Buckets must have globally unique name (名字必须独特)
    • Buckets are defined at region level

  • S3 Objects:
    • Objects (files) have a Key
    • The key is the FULL path (prefix + object name)
    • Max Object Size is 5TB
    • 如果上传超过 5 GB，就要用 Multi-Part Upload

S3 Bucket Policy

  • User-Based:
    • IAM Policies: which API calls should be allowed for a user from IAM
  • Resource-Based:
    • Bucket Policies: bucket rules (让 Object Public)
    • Object Access Control List (ACL)
    • Bucket Access Control List (ACL)
  • Encryption: encrypt objects in S3 with encryption keys

  • JSON based policies
    • Resoruces: buckets and objects
    • Effect: Allow / Deny
    • Actions: API to Allow or Deny
    • Principal: The account or user to apply the policy to

S3 Versioning

  • Enabled at the bucket level
    • Default is Null
  • Best pratice to version the buckets
    • 可以 roll back

S3 Replication (CRR & SRR)

  • CRR: Cross Region Replication (Compliance, lower latency access)
  • SRR: Same Region Replication (Create test environment)
    • Must enable Versioning
    • The Copying is asynchronous
    • Must give IAM permissions to S3
    • Only new objects are replicated, existing objects need S3 Batch Replication


S3 Storage Classes

  • 一共 7 种 Storage Class

  • S3 Standard - General Purpose
    • Used for frequently accessed data
    • Most common one

  • S3 Infrequent Access (IA)
    • For data is less frequent access but require rapid access when needed
    • Lower cost than Standard (但是需要 Retrieval Fee)

  • S3 Glacier Storage Classes
    • Low-cost object storage for archiving / backup
    • Price: storage + retrieval
    • S3 Glacier Instant Retrieval
    • S3 Glacier Flexible Retrieval
    • S3 Glacier Deep Archive - for long term storage

  • S3 Intelligent Tiering
    • Move objects automatically between Access Tiers based on usage
    • 就是自动帮你把 Object 移到不同的 Storage Class 里

11. AWS S3 Advance

S3 Lifecycle Rules

  • Transition Actions:
    • Configure objects to transition to another storage class
  • Expiration Actions:
    • Configure objects to expire (delete) after some time


  • S3 Analytics
    • Help you decide when to transition objects to the right storage class

S3 Requester Pays

  • The requester pays the cost of the request intead of the bucket owner

S3 Event Notifications

  • Use Case: Automatically react to certain event happened in S3 (SNS, Lambda)
    • Need to have IAM Permissions
    • All events will go through AWS EventBridge

S3 Performance

  • For Upload:
    • Multi-Part Upload: Parallelize uploads
    • S3 Transfer Acceleration: 可以和 Multi-Part Upload 一起用

  • For Download:
    • S3 Byte-Range Fetches: Parallelize GETs, Retrieve partial data

S3 Select & Glacier Select

  • Can do server-side filtering
  • 400% Faster, 80% Cheaper

S3 Batch Operations

  • Perform bulk operations on existing S3 objects
    • Encrypt all un-encrypted objects
  • Use S3 Inventory to get object list and use S3 Select to filter objects

12. AWS S3 Security

S3 Encryption

  • 一共有 4 种方法:
    • Server-Side Encryption (SSE)
      • Server-Side Encryption with Amazon S3-Managed Keys (SSE-S3) - Default
      • Server-Side Encryption with KMS Keys stored in AWS KMS (SSE-KMS)
      • Server-Side Encryption with Customer-Provided Keys (SSE-C)
    • Client-Side Encryption

  • SSE-S3: Enabled by default for new buckets & objects

  • SSE-KMS: User control + Audit key usage in CloudTrail
    • Limitation: Have API Call (对于 High Throughput 不友好)

  • SSE-C: AWS will not store the key, HTTPS must be used

  • Client-Side Encryption: User fully manages the keys and encryption cycle

  • Encryption in Transit (SSL/TLS)
    • 可以用 aws:SecureTransport 来 Enforce

S3 CORS

  • CORS: Cross-Origin Resource Sharing

  • S3 CORS
    • A client makes a cross-origin request on S3 bucket, need the correct CORS headers
    • Can allow specific origin or *

S3 MFA Delete

  • MFA Delete: 防止不小心删除文件
  • To use MFA Delete: Versioning must be enabled
  • Only the bucket owner can enable/disable MFA Delete

S3 Access Logs

  • Any request made to S3 will be logged to another S3 bucket
    • 可以作为 Data Analysis 的工具

S3 Pre-signed URLs

  • Have URL Expiration (过期)
    • 用在 Private Bucket 上，一段时间后 URL 过期

Glacier Vault Lock & S3 Object Lock

  • S3 Glacier Vault Lock
    • Adopt a WORM (Write Once Read Many)
    • The Object can’t be deleted

  • S3 Object Lock
    • Must have Versioning
    • Block an object version deletion for an amount of time
    • 两种 Mode: Compliance (所有人都没法删)，Governance (有些人可以删)
    • Legal Hold: 你自己可以删

S3 Access Points

  • Simplify security management for S3 Buckets
  • Each Access Point has:
    • DNS name
    • Access Point Policy

S3 Object Lambda

  • Use AWS Lambda to change object (在到达 Caller 之前)
  • 需要 S3 Access Point 和 S3 Object Lambda Access Points

13. CloudFront & AWS Global Accelerator

AWS CloudFront

  • Content Delivery Network (简单来说就是 CDN)
  • 提高 Read performance, Content is cached at the edge
  • 可以防止 DDoS protection (和 AWS Shield)
  • CloudFront Origins: S3 Bucket (OAC) 或者 Custom Origin (HTTP)
    • OAC: Origin Access Control

  • CloudFront vs S3 Cross Region Replication
    • CloudFront: 用于 static content
    • S3 Cross Region Replication (CRR): 用于 dynamic content

  • Price Classes: 不同地区的价格不一样
    • 可以用来 cost reduction (降低价格)

CloudFront Geo Restriction

  • Allowlist & Blocklist (针对国家进行 Access 限制)

CloudFront Cache Invalidation

  • 让 CloudFront 跳过 TTL 立即更新 (后端更新不代表 Cache 立即更新)

AWS Global Accelerator

  • 利用 AWS internal network to route application (更快)

  • AWS Global Accelerator vs CloudFront
    • CloudFront: Content is served at edge，比如图片和视频
    • Global Accelerator: Good for Non-HTTP or Static IP，比如游戏

14. AWS Storage Extra

AWS Snow Family

  • Data migration & Edge Computing (硬件形式)
    • Snowcone (体积小，适合特殊环境)
    • Snowball Edge
      • Snowball Edge Storage Optimized (80 TB of HDD)
      • Snowball Edge Compute Optimized
    • Snowmobile (不算在 Edge Computing 里, 最大的)

  • Edge Location: 比如海上或者矿洞里，没有网络的情况下
    • 所以需要 Edge Computing 和 Snow Family
  • AWS OpsHub: 用来管理 AWS Snow Family

Snowball into Glacier

  • Use S3 lifecycle policy (感觉啥不会就是 Lifecycle Policy)

AWS FSx

  • 3rd party high-performance file systems on AWS (第三方文件系统)
    • FSx for Windows (记这个就行)

  • 两种 Deployment Options:
    • Scratch File System: Temporary storage
    • Persistent File System: Long term storage

AWS Storage Gateway

  • Bridge between on-promise data and cloud data (Hybrid cloud)
  • Use cases: Disaster recovery, Backup & Restore

AWS Transfer Family

  • Use FTP protocol to transfer files (FTP, FTPS, SFTP)

AWS DataSync

  • Synchronize Data (Scheduled), Move large amount of data to and from:
    • On-promise and AWS S3, EFS, FSx 等等
    • File permissions and metadata are perserved
  • Snowcone 自带 DataSync

15. SQS, SNS, Kinesis, Active MQ

AWS SQS

  • 用来 Decouple Applications (比如处理视频)

  • 可以有很多 Consumer 来同时 (Parallel) 处理 Messages
    • Consumer delete messages after processing them

  • SQS 可以和 ASG (Auto Scaling Group) 一起用
  • SQS Message Visibility Timeout
    • 在 Timeout 期间 Message 不会被处理，Timeout 结束会重新回到 Queue
    • 如果遇到 “Process more than once”，那就增加 Visibility Timeout

SQS Long Polling

  • Decrease latency & Decrease API call (更好的 Performance)

SQS FIFO Queue

  • 按顺序传递 Message (SNS 也可以做到)

AWS SNS

  • Send one message to many receivers (subscriptions)
    • Publisher & Subscriber 模型 (一对多模型)
    • 可以给用户 Send Message

Fan Out Pattern

  • Push once in SNS, Receive in all SQS
  • 可以用于 Message Filtering (根据不同的 Filter Policy)

AWS Kinesis

  • 处理 streaming data in real-time

AWS Kinesis Data Streams

  • Have the ability to reprocess (replay) data
  • Once data is inserted in Kinesis, it can’t be deleted (immutability)
  • Data share the same partition goes the same shard (ordering)

  • Capacity Mode:
    • Provisioned mode: Choose a number of shards provisioned, scale manually
    • On-demand mode: No need to provision or manage capacity

AWS Kinesis Data Firehose

  • Fully Managed Service, automatic scaling, serverless
  • Near Real Time (不是 real-time)

  • Kinesis Data Streams vs Firehose
    • Data Streams: Write custom code, Real-time, Have data storage
    • Firehose: Fully managed, Near real-time, Don’t have data storage

Data ordering for Kinesis

  • Same key (Partition Key) will always go to the same shard (有顺序)
  • 相比之下 SQS 只有 FIFO (针对于需要大量 Consumer)

SQS vs SNS vs Kinesis

AWS MQ

  • 管理第三方消息代理 (Message Broker) 软件
    • RabbitMQ and ActiveMQ

16. Containers on AWS

AWS ECS

  • Elastic Container Service (Manage Docker containers on AWS)
  • 两种 Launch Type: EC2 (Need Provision) 和 Fargate (Serverless, No Provision)

  • Fargate: Serverless (重点)

  • IAM Roles for ECS
    • EC2 Instance Profile: 针对 EC2 Launch Type
    • ECS Task Role: 每个 Task 都有自己的 Role (负责)

  • ECS 可以和 Load Balancer 一起用
  • ECS 可以和 EFS 一起用 (Fargate + EFS = Serverless, 管理文件)

ECS Auto Scaling

  • 自动 increase/decrease the number of ECS tasks
    • Step Scaling: scale based on CloudWatch Alarm
    • Scheduled Scaling: scale based on specific time

  • Elastic Container Registry (ECR)
    • Store and manage Docker images on AWS

    • Support 2 deployment mode: EC2 & Fargate
    • 针对已经使用 Kubernetes 的

  • Leverages a Container Storage Interface (CSI) compliant driver

  • EKS Node Type (一共三种)
    • Managed Node Groups (针对 EC2 Instance)
    • Self-Managed Nodes (自己创建)
    • AWS Fargate (不需要管理)

AWS App Runner

  • Deploy web application and APIs (自动 Build 和 Deploy)
  • No infrasture experience required (不需要 AWS 经验)
