2022 HireEZ 实习

Jira 工单生成器

为 BI team 开发了一个自动化工单创建工具，替代原来手动从 Excel 复制粘贴到 Jira 的流程。工具支持树状结构批量创建 parent/child ticket，大幅减少重复性工作

  • 前端用 React 构建，渲染树状工单结构
  • Consume AWS Lambda 返回的员工信息数据
  • 调用 Jira REST API 批量创建工单
  • 为 BI team 节省每周数小时的手动操作时间

技术栈: React, AWS Lambda, Jira REST API

GA4 Tracking & 测试

在现有产品上集成 Google Analytics 4 埋点，追踪用户行为数据。用 Playwright 写 UI 测试验证埋点数据采集的准确性

  • GA4: 埋点追踪用户行为数据（页面浏览、按钮点击、功能使用等），为产品决策和 UX 优化提供数据支持
  • Playwright: 写端到端 UI 测试，模拟用户操作验证埋点事件是否正确触发和采集

技术栈: Google Analytics 4, Playwright

2024 HireEZ 实习

Chrome Extension Candidate Matching

为公司 Chrome extension 添加候选人匹配功能，将原本只在 web 端有的功能移植到 extension

  • 用 content script 解析页面，从 Greenhouse URL 和 LinkedIn 页面提取候选人 ID
  • 支持单个 profile 页和 LinkedIn My Network 页面批量匹配
  • 调用现有后端 API，匹配数据库中具有相同经验或技能的候选人
  • 用 React + 公司组件库构建展示组件

技术栈: Chrome Extension, React, JavaScript, HTML, CSS

AWS Q Business 知识检索系统（Research）

研究使用 AWS Q Business 搭建内部知识库系统，通过 Low-Code 方式连接公司的 Jira、Confluence 和 Slack 作为数据源

  • 用 AWS Q Business 的 connector 整合多个内部工具的文档和数据
  • 用 CloudFormation 写 IaC template，方便重复部署 (ref: https://www.youtube.com/watch?v=hX0FHWnvyKY)
  • 产出 demo 和研究文档给团队
  • 因 Q Business 当时刚发布功能较基础，最终作为可行性研究

技术栈: AWS Q Business, CloudFormation, Jira, Confluence, Slack

GitHub Actions CI/CD

研究并实现 GitHub Actions 自动化流程，用于前端项目的依赖版本检查

  • 写 Shell 脚本检查 package.json 依赖版本一致性
  • 集成到公司的 GitHub Actions 公共 repo，供后续新项目复用

技术栈: GitHub Actions, Shell
