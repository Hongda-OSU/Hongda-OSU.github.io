知识点

1. Describe Cloud Computing

Shared Responsibility Model

  • On Premise: All yours
  • Cloud Virtual Machine Responsibility: Mixed
  • Cloud App Service Responsibility: Mixed / Shared
  • Cloud SaaS Responsibility: Mixed / Shared

  • General Availability (GA) mode 代表 Anyone can use the service for any reason
    • Most Azure GA service have Service Level Agreement (SLA)
  • What happens if Azure does not meet its own Service Level Agreement guarantee (SLA)
    • Ans: A discount will be applied to the customer’s Azure bill, which is a compensation for an under-performing Azure product or service
  • Azure Free Account: default is 200 $
  • Azure Service 会经过 Private Preview, Public Preview 和 General Availability
  • SLA 控制 public cloud 的 high availability

Public / Private Preview mode

  • Public Preview mode
    • Anyone can use the service but normal service level agreement do not apply
  • Private Preview mode
    • You have to apply to get selected in order to use that service

Public Cloud, Private Cloud, Hybrid Cloud

  • Public Cloud: Azure owns the hardware (类似 Azure 和 AWS 这种)
  • Private Cloud: 一般是 Goverment 专用
  • Hybrid Cloud: Combination of public and private clouds

Cloud Pricing

  • 55 + Free services
  • Pay for Time: Charged by time used
  • Pay for GB: Pay for data storage used
  • Pay for Operations: Charged per operation (read, write, list, delete)

2. Benefits of Cloud Computing

High Availability Benefit

  • Ability of a system to remain operational to users during planned or unplanned outages

Scalability Benefit

  • Ability of a system to accommodate increasing demand by adding or removing resources as needed

  • Vertical Scaling
    • Also called “scaling up” or “scaling down”
    • Adding more resources to a single server

  • Horizontal Scaling
    • Also called “scaling out” or “scaling in”
    • Adding more servers to a system

Elasticity Benefit

  • Ability of a system to quickly and easily scale up or down the amount of resources that a system uses in response to changing demand (例如: Auto Scaling)

Reliability Benefit

  • Ability of a system to be accessible and usable by users when they need it
  • Auto-scaling, Multi-region deployments, Data backup and replication

Predictability Benefit

  • Ability to forcast and control the performance and behavior of a system

Security Benefit

Governance Benefit

  • The process of defining, implementing, and monitoring a framework of policies that guides an orgnaization’s cloud operations (例如: Auditing & Reporting)

Managability Benefit

  • Management of the Cloud
    • Templates, Automation, Scaling

  • Management in the Cloud
    • Web portal, CLI, APIs

3. Cloud Service Types

Cloud Service Types

  • Infrastructure as a Service (IaaS)
    • Computing, Storage, Networking
    • 例子: Azure Virtual Machines, Azure Storage, Virtual Networking

  • Platform as a Service (PaaS)
    • Service layer on top of IaaS computing, storage, networking
    • Complete development and deployment environment in the cloud
    • 例子: Azure App Services, Azure SQL Database, Azure Front Door

  • Software as a Service (SaaS)
    • Cloud apps (例子: Google Drive)

Serverless

  • Azure Functions, Container Apps, SQL Database, Cosmos DB
  • Logic Apps, Functions, Service fabric 属于 Serverless Model

4. Core Architectural Components of Azure

Regions, Region Pairs, Sovereign Regions

  • Regions: Areas of the world where Azure has a set of datacenters (minimum 3)
    • Azure 原来有 54 个 region, 现在有 60 + region
  • Region pairs: Have highest speed connections and special treatment during Azure updates

  • Sovereign Azure: Not connected to the Azure Public Cloud
    • 例子: Azure US Goverment, Azure China

Availability Zone and Data Centers

  • Not every region supports Availability Zone
  • Zonal Services
    • You can choose a specific AZ to deploy the service to

  • Zone-Redundant Services
    • Automatically deployed across zones for you

  • Always Available Services
    • These are global services and Microsoft takes care of them

Resources and Resource Groups

  • Resources 到 Resource Groups 到 Subscriptions 到 Management Groups
  • Resources represent the Azure service that you have access to, such as a specific Virtual Machine, Storage Account, or Database

  • Resource Group: A logical grouping of resources
    • When need to delegate permissions to several Azure VMs simultaneously
    • Resource Group is a folder structure in Azure in which you organize resources like database, virtual machines, virtual networks, or almost any resource

Subscriptions

  • The billing unit within Azure (每个 department 都可以有自己的 subscription)

Management Groups

  • Management Groups 用来管理 Subscriptions

5. Azure Compute and Networking Services

Azure Compute services and Azure Virtual Machines

  • Technical Pillars of Azure

    • Compute services, Networking services, Storage services, Database services

  • Compute Types in Azure

  • Virtual Machines: Like a “server”, but it’s virtual
    • A “slice” of a physical machine shared with other customers

  • Availability Sets

    • Physically and logically separates virtual machines so that if one goes down, the others do not go down

  • Azure Virtual Machine Scale Sets

    • Can manage groups of virtual machines as one unit
    • Deploy and manage a set of identical virtual machines

  • Service Endpoints

    • Service endpoints are used to expose Azure services to a virtual network, providing communication between the two

Scaling Virtual Machines using VM Scale Sets (VMSS)

  • Increase the size of a VM (4 vCPU VM to 8 vCPU VM), this is scale up
  • Add more VMs and have them work together to handle the work, this is scale out

  • Virtual Machine Scale Sets (VMSS)
    • A group of virtual machines that can grow and shrink in quntity based on a predefined rule
    • Two or more virtual machines running the exact same code
    • With a “load balancer” in front to direct traffic randomly to one of the machines

App Services, Containers and Azure Virtual Desktop

  • App Services
    • Give your code and configuration to Azure, and they will run it
    • App Services is PaaS, App Services promise of performance but no access to hardware

  • Container Services
    • Contain everything the app needs to run in a “container image”
    • 例子: Azure Container Instance (ACI), Azure Container Apps, Azure Kubernetes Service (AKS)

  • Azure Virtual Desktop
    • Desktop version of Windows that runs in cloud

Azure Functions

  • Small pieces of code that run entirely in the cloud (类似 Lambda)
  • Azure Functions 不是 IaaS

Azure Networking services

  • Sometimes called VNets
  • By default, two virtual machines in Azure are not allowed to talk with each other
  • Virtual Networks are assigned an address space of either IPv4 or IPv6, or both

  • Subnets: All VNets are subdivided into one or more subnets
    • Best way to protect subnet is Network Security Group

  • Virtual Machines
    • All virtual machines must belong to at least one subnet, using a Network Interface Card (NIC)
    • 如果需要 minimize cost 但是 perserve associated hard disk and data, 可以去 deallocate the virtual machine when it’s not needed
    • Virtual Machine 需要 Virtual Network, Network Interface Card (NIC), Storage Account
    • Azure Virtual Machine 在一个 Scale Set 里最多有 1000 个

  • Network Security Group: Also called NSG
    • Access control list (ACL) that blocks traffic inbound and outbound from a subnet unless it matches certain rules

Network Peering, Azure DNS, VPN Gateway

  • Peering: Connect two subnets together

  • Azure DNS: Give IP addresses names using DNS

  • Azure VPN Gateway
    • Allow communication between a workstation and a network, or between two networks
    • Encrypts traffic between those two points
    • 如果需要 establish Site-to-Site VPN connection with Azure, on-premise 需要 a compatible VPN Gateway device

  • VPN Peering: Connect two distant networks using VPN devices

  • ExpressRoute
    • A private connection from your ISP to an Azure endpoint (high speed connection)

6. Azure Storage Services

Overview of Azure Storage

  • Azure Storage Services
    • Container (Blob) Storage, Disk Storage, Azure File Storage, Storage tiers
    • The maximum amount of Azure Storage space in virtually unlimited

  • Azure Storage (GPv2)
    • Subdivided into four types of data can store: Container, File, Queue, Table
    • Can hold up to 5 Petabytes

  • Premium Storage Options
    • Blob storage - can only hold containers (blobs)
    • Use premium SSD, More expensive

Container (Blob) Storage

  • Binary Large Object (BLOB): File like PDF, ZIP, CSV (类似 AWS S3)
  • Location
    • Keep data close to the person / service consuming it

  • Redundancy
    • Azure keeps 3 copies of your data by default
    • Locally or Zone redundant
    • 注意 Locally Redundant Storage (LRS) 是最便宜的

  • Global Redundancy
    • Azure keeps 6 copies of your data
    • 3 locally, 3 in another region of the “geo”

  • Access Tiers
    • Hot: the default, balanced access
    • Cool: cheaper storage with more expensive read / write operations
    • Cold: much cheaper storage, more expensive read / write operations
    • Archive: cannot get immediate access to files, cheapest storage, most expensive operations

AzCopy

  • AzCopy 是微软提供的命令行工具，用于在本地存储和Azure存储服务之间高效地传输数据。用户可以使用AzCopy在Azure Blob存储、文件存储和表存储之间进行数据的上传、下载和复制操作。它支持批量传输、并行处理和断点续传，适用于大规模数据迁移和备份。

Azure Files and Azure File Sync

  • Azure File
    • True hierarchical structure with folders
    • Replace or supplement on-premise file storage

  • Azure File Sync
    • Hybrid option: on-premise files with cloud option
    • Cannot used to move individual files back and forth between Clouds

Azure Migrate

  • Azure Migrate 是一个微软提供的服务，用于帮助用户评估和迁移本地数据中心的工作负载到Azure云平台。它提供工具和服务来评估应用程序、虚拟机和数据库的迁移可行性，进行性能分析，并为迁移过程提供指导和支持。

Azure Data Box

  • Azure Data Box 是微软提供的一种数据传输解决方案，用于帮助用户将大规模的数据从本地数据中心迁移到Azure云。它包括一系列硬件设备，用户可以将数据复制到这些设备上，然后将设备运送到微软数据中心进行数据上传。这种方法适用于网络带宽不足以支持大规模数据传输的情况，提供了一种安全、高效的物理数据迁移途径。

  • 有 Data Box (100TB), Data Box Disk (8TB) and Data Box Heavy (1PB)

Azure Key Valut

  • The recommended way within Azure to store secrets such as private cryptographic keys

7. Identity, Access and Security

Identity and Microsoft Entra ID

  • Microsoft Entra ID（以前称为 Azure Active Directory，Azure AD）是微软的云端身份和访问管理服务。它为用户提供身份验证、访问控制和目录管理功能，帮助企业保护和管理用户身份，并确保只有授权人员能够访问资源。Entra ID 支持单点登录（SSO）、多因素身份验证（MFA）、自助密码重置等功能，广泛用于应用程序、服务和设备的安全访问管理。
  • Microsoft Entra ID 提供 Authentication 和 Application management
  • Microsoft Entra ID licenses 有 Microsoft Entra ID P1 和 Microsoft Entra ID Free
  • Conditional Access
    • Ensure a user can only access application from compliant devices
    • Ensure a user can only access Microsoft Office 365 from apporved client application

Authentication vs Authorization

  • Authentication: user provide who they are (user id and password)
  • Authorization: ensure a user is permitted to perform an action

Azure AD Conditional Access

  • Azure AD Conditional Access 是一种安全工具，用于根据用户身份、设备和位置等条件，动态控制对企业资源的访问。它帮助确保只有符合特定条件的用户和设备才能访问敏感数据和应用。

Multi-Factor Authentication (MFA)

  • Require 2 or more pieces of evidence in order to log in

  • SSO 是 the ability to log in once and use the existing userid and password to sign in other applications, and not have to create / memorize a new one

Passwordless

  • Alternative to MFA
  • Using a PIN or biometric recognition with Windows devices

Role-Based Access Control (RBAC)

  • Microsoft preferred solution for authorization (就是 AWS 那一套)
    • Create roles for your organization
    • Principle of Least Privilege

Zero-Trust Model of Security

  • Can’t trust any connection regardless where it comes from
  • Force everyone to prove their identity
  • The core principle of Zero Trust is: Assume your network has been breached already

Microsoft Defender for Cloud

  • Microsoft Defender for Cloud 是一款云安全管理工具，用于保护混合云环境中的资源。它提供威胁防护、漏洞管理和安全合规性监控，帮助识别和应对潜在的安全威胁，确保云资源的安全。

Azure Firewall

  • Restrict traffic to multiple virtual networks in multiple subscriptions with Azure firewall
  • Azure Firewall 是一个云原生的网络安全服务，用于保护Azure虚拟网络中的资源。它提供高可用性和可伸缩的防火墙功能，支持基于规则的流量控制、威胁情报集成和日志记录，确保网络流量的安全性。

ISO

  • ISO means International Organization for Standardization

Public IP

  • Public IP enables access to your data or application from outside the virtual network

8. Cost Management in Azure

Factors that Affect Cost

  • One of the benefits of cloud computing is the flexibility in pricing options
  • Factor that affect cost: Time, Consumption, Service Tier 等等
  • For tax optimization, OpEX (Operating Expenditure) is preferable

Azure Pricing Calculator

  • Azure Pricing Calculator 是一个在线工具，帮助用户估算在Azure云平台上使用各种服务的成本。用户可以根据需要选择不同的服务和配置，实时查看价格，并根据预算进行规划和优化。

Total Cost of Ownership Calculator (TCO)

  • Total Cost of Ownership Calculator (TCO) 是微软提供的一个工具，用于帮助企业评估将本地数据中心迁移到Azure云的总体成本。TCO计算器考虑了硬件、软件、运营和维护等多个方面的费用，帮助用户比较云端和本地部署的长期成本和节省。
  • 注意, Total Cost of Ownership 同时包括了诸如 Real estate costs, Salary 这些东西

Azure Cost Management

  • A free tool inside Azure Portal to analyze spending
  • Tracking against Budget (比如超过多少钱会 alert)
  • Azure Cost Management 是一个工具集，用于帮助用户监控、管理和优化在Azure上的云支出。它提供费用分析、预算设定、成本分配和优化建议，帮助企业控制云资源成本，确保支出在预算范围内。
  • Azure Cost Management 用来 Create and Manage budgets 和 Generate historical reports and forecast future usage

Resource Tags

  • Resource Tags 是Azure中的一种标记机制，允许用户为Azure资源分配自定义标签（键-值对）。这些标签用于组织、分类和管理资源，便于进行成本分配、权限管理和资源查找。通过使用标签，企业可以更有效地管理和优化其云资源。

9. Azure Governance and Compliance

Azure Governance and Compliance

  • Governance, within a company, are the leaders / processes that sets the rules
  • 例子: Azure Policy, Resource Locks, Microsoft Purview, RBAC

Azure Policy

  • Create rules for some or all Azure resources and groups (类似 IAM Policy)
  • Create custom policy using JSON
  • Azure Policy 是一个管理和强制实施资源合规性的工具。它允许用户定义和应用规则，确保Azure环境中的资源符合企业标准和法规要求。通过Azure Policy，用户可以评估现有资源的合规性、自动纠正违规行为，并保持一致的资源配置和管理。
  • Azure Policy 可以 prevent certain Azure Virtual Machine instance types from being used in a resource group

Azure Blueprints

  • To define a repeatable set of Azure resources that implements and adheres to an organization’s standards, patterns, and requirements
  • Azure Blueprints allows you to create new subscrption that already have policies, roles, resource group, and ARM templates (重要)

Resource Locks

  • Types of Resource Locks: Read Only, Can Not Delete
  • Using RBAC, you can restrict who can unlock
  • Resource Locks 是Azure中的一种功能，用于防止对关键资源的意外更改或删除。它们有两种类型：只读锁（Read-only）和删除锁（Delete）。只读锁限制所有对资源的修改，而删除锁则仅限制删除操作。通过使用资源锁，可以提高资源的安全性和稳定性。
  • “read only” resouce lock 可以让 storage account cannot have its properties altered but it doesn’t affect the data itself

Microsoft Purview

  • Microsoft Purview 是一款数据治理和合规性解决方案，旨在帮助企业管理其数据资产。它提供数据发现、分类、编目和追踪等功能，确保数据在整个组织内得到正确的管理和使用。Purview帮助企业遵守法规要求，提高数据的可见性和控制，促进数据的安全和合规。

10. Tools for Managing and Deploying Azure Resources

Azure Portal and Command Line Tools

  • Azure Portal 是一个基于网页的界面，用户可以通过它管理和监控Azure资源。它提供了直观的仪表板和可视化工具，使用户能够轻松地创建、配置和管理云服务。
  • Command Line Tools 包括 Azure CLI 和 Azure PowerShell，这些工具允许用户通过命令行创建、管理和自动化Azure资源。
    • CLI 和 PowerShell 是为了 Automation 的

Azure Arc

  • Cross-Platform VM & Container Management
  • Manage Windows and Linux physical servers and VM outside of Azure

Infrastructure as Code (IaC)

  • Define your desireed infrastructure in a configuration file

ARM Templates

  • The deployment and management service for Azure
  • ARM 是 Azure Resource Manager

11. Monitoring Tools

Azure Advisor

  • Azure Advisor 是一个个性化的云咨询服务，它提供最佳实践建议，帮助用户优化Azure资源的配置和使用。Advisor根据用户的资源使用情况，提供在高可用性、安全性、性能、成本管理和操作卓越性方面的改进建议，帮助用户提高Azure环境的整体效率和安全性。
  • 5 点: Cost, Security, Reliability, Operational Excellence, Performance
  • Azure Advisor give specific recommendations based on actual usage

Azure Service Health

  • Azure Service Health 是一个工具，用于监控和报告Azure服务的健康状况和事件影响。它提供关于Azure服务中断、计划内维护和其他重要事件的实时通知和详细信息，帮助用户了解和管理其Azure资源的可用性和性能。通过Azure Service Health，用户可以及时采取措施，减轻潜在问题对其业务的影响。
  • 提供 alert 的

Azure Monitor

  • Azure Monitor 中央仪表板是一种集中的监控和管理工具，用于汇总和可视化来自各种Azure资源的数据。它允许用户在单一视图中查看关键指标、日志和诊断信息，帮助监控应用程序性能、检测异常行为和排查问题。通过自定义仪表板，用户可以实时跟踪资源的健康状况和性能，从而优化运维和管理工作。
  • Azure Mointor 可以监控 on-premise servers 的 performance
  • 可以 collect logs 然后集中展示在一个 central dashboard, 这个 dashboard 可以跑 query
  • Azure Monitor 可以用来 trigger autoscaling

Azure Monitor Reports

  • Azure Monitor 报告（Reports）是用于生成和查看关于Azure资源性能和健康状况的详细报告的工具。通过Azure Monitor，用户可以创建自定义报告，包含关键指标、日志和警报信息。这些报告帮助用户深入了解资源的使用情况、性能趋势和潜在问题，从而进行更有效的优化和决策。

DDoS Protection

  • Two types of DDoS protection services Azure provide
    • DDoS Network Protection
    • DDoS IP Protection
  • Azure DDoS Protection 不保护 Application (L7) level attacks

Azure Security Center

  • Provide a security dashboard that contains all the security and threat protection in one place

Azure Cognitive

  • Contains pre-built machine learning models
  • Azure Cognitive Services API can do
    • Speaking text in an extremely realistic way
    • Recognize text in an image
    • Create text from audio
    • Translate text from one language to another

Azure DevOps

  • Designed to help companies get their in-house developed code from the code repository, through automated unit testing, and onto Azure using a service called Pipelines

Azure Application Insights

  • Automatically detect performance anomalies for web apps

Log Analutics Workspace

  • Collect log and metric data from various Azure Resources

HDInsight

  • Host the Apache Hadoop big data analysis tools inside Azure

Business Continuity / Disaster Recovery (BC / DR)

  • The concept of being able to get your applications and data running in another environment quickly

Policy Initiative

  • The ability to group policies together
